# Module 1: Google Cloud Computing Foundations — Cloud Computing Fundamentals

## Summary of the Module
This course serves as the introductory track of the Google Cloud Computing Foundations series. It provides a formal architectural analysis of cloud computing paradigm shifts, contrasting traditional bare-metal infrastructures against multi-tenant cloud deployments. The curriculum covers resource hierarchy mechanics, programmatic environment interfaces (Google Cloud SDK & Cloud Shell), and a comprehensive operational evaluation of Google Cloud’s primary compute options—spanning Infrastructure as a Service (IaaS), Platform as a Service (PaaS), Containers, and Serverless architectures.

## Module Objectives
* **Architectural Decoupling:** Articulate the design shifts between rigid traditional infrastructure and highly elastic, shared-responsibility cloud models (IaaS, PaaS, SaaS).
* **Resource and Identity Isolation:** Analyze the global resource organization layer (Projects) and implement state-mutation policies via Cloud IAM and service APIs.
* **Compute Plane Evaluation:** Evaluate, provision, and deploy cloud resources across Google Compute Engine, App Engine, Cloud Run functions, Google Kubernetes Engine (GKE), and Cloud Run.
* **Programmatic Environment Interfacing:** Formulate and execute infrastructure deployment scripts utilizing the Google Cloud SDK (`gcloud`) within the ephemeral Cloud Shell sandbox.

## Technical Architecture Track

The course is organized into structured progressive tracks designed to systematically scale engineering competencies:

### Track 1: Cloud Architecture Paradigms
* Detailed analysis of cloud computing abstractions.
* Structural comparison: **Cloud Infrastructure vs. Traditional On-Premises Architecture**.
* Operational boundaries of **IaaS, PaaS, and SaaS** delivery models.
* Exploration of the global Google Cloud physical and virtual network architecture.

### Track 2: Resource Management & Programmatic Interfaces
* Navigation of the Google Cloud Console plane and isolation of the **Billing and Project namespaces**.
* Local and cloud-hosted environment setup via **Google Cloud SDK** initialization.
* Interactive script orchestration using **Cloud Shell** and programmatic **Google Cloud APIs**.

### Track 3: Compute Options Exploration
* **Compute Engine (IaaS):** Virtual Machine (VM) provisioning, machine type allocation, and regional/zonal high-availability topologies.
* **App Engine (PaaS):** Highly scalable, zero-ops application deployment utilizing managed runtime stacks (Python).
* **Cloud Run Functions (Serverless Events):** Deployment of event-driven, micro-scale execution blocks via the command-line interface.
* **Google Kubernetes Engine (GKE):** Containerization, orchestration, and scaling of monolithic apps into container clusters.
* **Cloud Run (Serverless Containers):** Execution of stateless HTTP containers within a fully managed serverless infrastructure.
